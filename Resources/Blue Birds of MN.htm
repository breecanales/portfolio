Walkthrough

Content Map: Displayed in the 'Content Map' section of the About page.

Wireframe: Displayed in the 'Wireframes' section of the About page.

Home, About, Featured, & Contact Sections: Site is divided into these four sections.

Unique favicon: Displayed in the browser tab.

CSS Glyphs, Icons or Sprites: I used icons throughout the website. I drew SVG icons for each bird which are displayed in the dropdown menu in the Featured page. There are icons displayed in the 'Credits' section for certain resources in the About page. I also used Glyphs to display the copyright symbol ©.

An instance of the video element: I included a video about the Audubon Society on the Contact page.

An instance of the audio element: I included audio clips of each bird on the Featured page. Citations can be found in the 'Credits' section.

An implementation of a slideshow: The Featured page uses a slideshow to display the various blue birds of Minnesota. The slideshow is controlled by left and right arrows that are situated on the left and right side of the bird information panel.

Web fonts: This site uses the Merriweather, Raleway, and Open Sans fonts from Google Fonts.

An instance of the canvas element: I used a canvas element in the Admin page to display the monthly email list signups.

An instance of the SVG element: There are two instances of SVG elements in this website: the bird silhouettes found on the Featured page, and the white bird icon found in the website's navigation bar.

HTML5 form input validation: On the Contact page, the form requires that a person fills out at least their first name and provide a valid email before they are allowed to click the submit button.

JSON for data interchange: On the Featured page, the link to the bird photo and its information is all stored in a JSON file (bird-info.json). When someone changes the bird they are viewing, this information is loaded from the JSON file to the page. I also used JSON on the Contact page for form submission to SQLite.

Footer navigation: Every page implements footer navigation.

Server-side persistent storage: When someone fills out the contact form on the Contact page, their information is stored in a SQLite database. This information can only be seen on the Admin page.

Data visualization: On the Admin page, I used Chart.js to display the monthly email list signups as a line graph.

An instance of a Custom Elements: On the Contact page, I used the Google Maps custom element to display the meeting location of the St. Paul chapter of the Audubon Society.

Include an implementation of authentication: This site has an Admin page where you can see all the emails that have been collected. There is a required username and password to be admitted to this page. The credentials are username=bluebirds and password=ilikebirds.

About page includes a Credits section that lists/links all external resources: Displayed in the 'Credits' section of the About page.
